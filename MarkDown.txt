Спецификация требований
=======================

Введение
--------

Документ содержит спецификацию требований для проекта “Онлайн-сервис пиццерии “Большой папа”, предназначенный для заказа и доставки пиццы. Документ в целом соответствует рекомендациям стандарта IEEE-29148 Requirements Engineering с точки зрения как структуры, так и содержания.

В документе используются диаграммы, поясняющие его текст, созданные в соответствии с нотациями UML и BPMN.
Документ предназначен для всех заинтересованных сторон, участвующих в создании проекта.

Описание системы
----------------

Назначение
----------

Система предназначена для автоматизации процесса создания и обработки онлайн заказов на изготовление и доставку пиццы в указанный в заказе пункт назначения.

Границы системы и бизнес-требования
-----------------------------------
“Онлайн-сервис пиццерии “Большой папа”” реализуется системой, состоящей из бекенда и приложения — веб-сайта, предоставляющего пользовательский интерфейс. Основные сценарии, реализуемые системой, показаны на Диаграмме 1.

Диаграмма 1.
![Диаграмма 1](diagramma.jpg "Диаграмма 1")

| Параметр | Тип данных | Описание |
|:----------|:------------:|:----------|
|qty|int|количество экземпляров позиции|
|pizza_id|int|Уникальный идентификатор пиццы|


Пример ```json``` тела запроса:
```json
{
	{
	“pizza_id”:45,
	“qty”:3
	},
	{
	“pizza_id”:67,
	“qty”:2
	}
}
```
